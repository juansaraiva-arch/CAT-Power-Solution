1. Application Overview

CAT QuickSize is an engineering tool for sizing prime power generation solutions for data centers. It calculates the optimal fleet of CAT natural gas generators, Battery Energy Storage Systems (BESS), emissions controls, and full financial analysis.

1.1 Architecture

ComponentTechnologyDescription
Pilot UIStreamlitInteractive web interface with sliders, tables, and charts
REST APIFastAPI (Python 3.11)8 routers, endpoints for sizing, generators, projects
Web FrontendReact + TypeScriptModern interface for BDMs (Business Development Managers)
Calculation EnginePython (core/engine.py)14 stateless functions — intellectual property core
LibraryPython (core/generator_library.py)8 generators with ~20 specifications each
PipelinePython (sizing_pipeline.py)23-step sequential calculation orchestration

1.2 Calculation Flow

The user enters load parameters, site conditions, technology choices, and economic inputs. The pipeline executes 23 sequential steps that produce: optimal fleet sizing, BESS requirements, electrical stability analysis, emissions assessment, physical footprint, and full financial analysis including LCOE, NPV, and payback period.

Note: All engine functions are stateless: they receive inputs and produce pure outputs. No UI, database, or session dependencies.

4.2 Transient Stability transient_stability_check()

Critical voltage sag check for AI workloads. A sag > 10% can cause GPU server resets.

# Parameters:
xd_pu    = generator subtransient reactance (p.u.)
num_units = number of parallel generators
step_load_pct = instantaneous load change (%)

# Calculation:
equiv_xd = xd_pu / sqrt(num_units)
voltage_sag = (step_load_pct / 100) × equiv_xd × 100

# Criterion: PASS if voltage_sag ≤ 10%

Output: tuple(bool, float) — (passes_check, sag_percentage)

4.3 Frequency Screening frequency_screening()

Analytical frequency nadir and ROCOF (Rate of Change of Frequency) screening using simplified swing equation. Not a substitute for full ODE simulation.

# Model constants:
R = 0.05     # 5% droop
D = 2.0      # Damping (data center PE loads)
T_gov = 0.5   # Governor time constant (s)
pf_gen = 0.85 # Generator power factor

# Total system MVA:
S_gen_mva = unit_cap_mw / pf_gen
S_total_mva = n_running × S_gen_mva

# Total inertia (including BESS virtual if applicable):
H_total = H_mech + H_bess
# H_bess = 4.0 × min(1.0, bess_ratio / 0.2) if BESS enabled

# ROCOF (Rate of Change of Frequency):
ROCOF = (P_step_pu × freq_hz) / (2 × H_total)

# Frequency nadir:
Δf_ss = (P_step_pu / (D + 1/R)) × freq_hz
overshoot = 1.0 + sqrt(T_gov / (4 × H_total))
Δf_nadir = Δf_ss × overshoot
nadir_hz = freq_hz - Δf_nadir

# Criteria:
nadir_ok = nadir_hz ≥ 59.5 Hz (60 Hz) or 49.5 Hz (50 Hz)
rocof_ok = ROCOF ≤ 1.0 Hz/s

Output: dict with nadir_hz, rocof_hz_s, nadir_ok, rocof_ok, H_total, P_step_pu, notes.

4.4 Spinning Reserve calculate_spinning_reserve_units()

Calculates the number of running generators needed considering spinning reserve requirements. Behavior differs based on BESS availability.

# Required spinning reserve:
spinning_reserve_mw = p_avg_load × (spinning_reserve_pct / 100)

# WITHOUT BESS: generators must provide ALL reserve as headroom
required_online_cap = p_avg_load + spinning_from_gens

# WITH BESS (if covering ≥90% of reserve):
required_online_cap = p_avg_load × 1.05

# Units needed:
n_units_running = ceil(required_online_cap / unit_capacity)

Output: dict with n_units_running, load_per_unit_pct, spinning_reserve_mw, spinning_from_gens, spinning_from_bess, total_online_capacity, headroom_available.

4.5 BESS Sizing calculate_bess_requirements()

Sophisticated BESS sizing based on actual transient analysis. Evaluates 6 independent components and selects the maximum.

The 6 Sizing Components:

#ComponentFormula
1Step Load Supportmax(0, step_load_mw - gen_step_mw)
2Peak Shavingp_peak - p_avg
3Ramp Rate Supportmax(0, (load_ramp_req - gen_ramp_rate) × 10)
4Frequency Regulationp_avg × 0.05 (5%)
5Black Startp_peak × 0.05 (if enabled)
6Spinning Reservep_avg × (step_load_pct / 100)
# Total BESS power = max of all 6 components, floor at 15% of peak
bess_power = max(comp1, comp2, comp3, comp4, comp5, comp6, p_peak × 0.15)

# Energy at C-rate = 1.0 and 85% DoD:
bess_energy = bess_power / 1.0 / 0.85

Output: tuple(float, float, dict) — (bess_power_mw, bess_energy_mwh, component_breakdown)

4.6 BESS Reliability Credit calculate_bess_reliability_credit()

Calculates how many genset equivalents the BESS can replace for reliability purposes. BESS realistically covers 2–4 hours (bridge power while backup starts), NOT the full 48-hour MTTR.

realistic_coverage_hrs = 2.0
power_credit = bess_power_mw / unit_capacity_mw
energy_credit = bess_energy_mwh / (unit_capacity_mw × realistic_coverage_hrs)

raw_credit = min(power_credit, energy_credit)

# Conservative factors:
bess_availability = 0.98
coverage_factor = 0.70

effective_credit = raw_credit × bess_availability × coverage_factor

Output: tuple(float, dict) — (effective_credit, credit_breakdown)

4.7 Availability (Weibull / Binomial N+X) calculate_availability_weibull()

Reliability model using industry-standard availability formula including planned maintenance. Calculates system availability and its evolution over time with aging.

# Unit availability (including planned maintenance):
annual_maint_hrs = (8760 / maint_interval) × maint_duration
total_unavailable = MTTR + annual_maint_hrs
A_unit = MTBF / (MTBF + total_unavailable)

# System availability — binomial N+X model:
A_system = Σk=N_runN_total C(N_total, k) × Aunitk × (1 - Aunit)(N_total - k)

# Aging over project life:
aging_factor(year) = max(0.95, 1.0 - year × 0.001)
A_unit_aged(year) = A_unit × aging_factor(year)

Output: tuple(float, list[float]) — (year1_availability, availability_curve_per_year)

4.8 Fleet Optimization optimize_fleet_size()

Multi-objective optimization that finds the optimal number of running generators, balancing capacity, efficiency, and step load coverage. Optimal load point is 72.5%.

# Optimal efficiency load point:
optimal_load = 72.5%

# Candidates evaluated: n_min-1 through n_min+2
# Only options where 30% ≤ load_pct ≤ 95% are considered

# Multi-objective score per candidate:
load_penalty = |load_pct - 72.5| / 100
score = efficiency × (1 - load_penalty × 0.5)

# Selection: n with highest score

Output: tuple(int, dict) — (optimal_n, evaluated_options)

4.9 MACRS Depreciation calculate_macrs_depreciation()

MACRS 5-year accelerated depreciation schedule (US tax code). Calculates present value of tax shield benefit.

# MACRS 5-year schedule:
rates = [20.0%, 32.0%, 19.2%, 11.52%, 11.52%, 5.76%]
tax_rate = 0.21 # US corporate rate 21%

PV_benefit = Σ (CAPEX × rateyr × tax_rate) / (1 + WACC)yr

Output: float — present value of total tax shield benefit.

4.10 Noise Propagation (3 functions)

Acoustic propagation model for N identical sources with acoustic treatment.

4.10a calculate_combined_noise()

L_total = (L_source - attenuation) + 10 × log10(N)

4.10b noise_at_distance()

# Point source, hemispherical spreading:
L_receiver = L_source - 20 × log10(distance_m) - 11

4.10c noise_setback_distance()

d_min = 10((L_combined - L_limit - 11) / 20)

4.11 Site Derating calculate_site_derate()

Automatically calculates generator derating factor based on site conditions. Three multiplicative factors:

# Temperature (CAT generators do not derate until 40°C):
temp_derate = 1 - ((T - 40) / 5.5) × 0.01   if T > 40°C, else 1.0

# Altitude (no derating until 300m):
alt_derate = 1 - ((alt - 300) / 300) × 0.035   if alt > 300m, else 1.0

# Fuel quality:
fuel_derate = 1 - ((80 - MN) / 100) × 0.15   if MN < 80, else 1.0

# Combined factor (floor at 0.50):
derate = max(0.50, temp_derate × alt_derate × fuel_derate)

4.12 Emissions calculate_emissions()

Complete emissions engine with 5 regulatory jurisdictions, aftertreatment recommendations, urea/DEF sizing, and compliance matrix.

Primary Calculations

# Emissions in lb/hr (industry formula for gas engines):
NOx_lb_hr = p_avg_kw × (NOx_g_kwh / 1000)
CO_lb_hr  = p_avg_kw × (CO_g_kwh / 1000)

# Annual emissions (tons/year):
NOx_tpy = (NOx_lb_hr × effective_hours) / 2000
CO2_tpy = fuel_input_mmbtu_hr × 0.0531 × 8760 × CF

# Regulatory conversions:
mg_Nm3 = g_kWh × 3.6 / 4.5
ppmvd = mg_Nm3 / factor (NOx: /2.05, CO: /1.25)

# Aftertreatment triggers:
needs_SCR = NOx_tpy > 100 OR NOx_g_kwh > 0.5
needs_OxiCat = CO_tpy > 100 OR CO_g_kwh > 2.0
SCR_reduction = 90%, OxiCat_reduction = 85%

Urea/DEF Sizing

urea_liters_per_MWh = 20.0 (if SCR required)
storage = daily_gallons × 30 (30-day autonomy)
num_totes = ceil(storage / 275) (standard IBC totes)
annual_cost = annual_gallons × $0.50 (bulk DEF)

4.14 LCOE / Financial calculate_lcoe()

Levelized Cost of Energy (LCOE), Net Present Value (NPV), and payback period calculation.

# Capital Recovery Factor (CRF):
CRF = (WACC × (1 + WACC)n) / ((1 + WACC)n - 1)

# Annualized CAPEX:
CAPEX_annual = total_capex × CRF

# LCOE:
LCOE = (CAPEX_annual + OM_annual + Fuel_annual + Carbon_annual) / MWh_annual

# NPV:
NPV = PV_savings + tax_benefits - total_capex - repowering_PV

# Simple payback:
Payback = total_capex / annual_savings

5. Sizing Pipeline — 23 Steps

The file sizing_pipeline.py (830 lines) orchestrates the 23 sequential steps that transform the 76 input variables into the 55+ output fields. Main function: run_full_sizing(inputs: SizingInput) -> SizingResult.

1
Resolve Generator — Loads selected generator from the library and applies user overrides if provided.
2
Load Calculations — p_total_dc = p_it × PUE, p_avg = p_total_dc × CF, p_peak = p_total_dc × peak_ratio. Applies distribution losses: p_at_gen = p_total / (1 - dist_loss%).
3
Site Derating — Auto-calculates or applies manual factor. unit_site_cap = unit_iso_cap × derate_factor.
4
BESS Sizing — If use_bess=true, executes calculate_bess_requirements() with 6 transient sizing components.
5
Spinning Reserve — calculate_spinning_reserve_units() determines minimum generators for headroom.
6
Fleet Optimization — optimize_fleet_size() finds optimal efficiency point at 72.5% load. Takes maximum between optimization result and spinning reserve requirement.
7
Availability Target — Converts avail_req from percentage to decimal. Sets MTTR = 48 hours.
8
Reliability Configurations A/B/C — Generates up to 3 configurations:
Config A: No BESS (baseline) — generators only with N+X reserves.
Config B: BESS Transient Only — BESS for transients, fewer generators needed.
Config C: Hybrid or Reliability Priority — additional BESS for reliability. Parameters by strategy: Hybrid covers 5 gensets / 2h, Reliability Priority covers 8 gensets / 2.5h. Includes BESS reliability credit (applied with conservative 0.65 factor).
9
Select Configuration — Based on bess_strategy: "Transient Only" selects Config B, "Hybrid"/"Reliability Priority" selects Config C, fallback selects Config A.
10
Extract Final Values — Extracts n_running, n_reserve, n_total, installed_cap, load_per_unit from selected config. Calculates BESS breakdown including reliability backup.
11
Fleet Efficiency + Site Corrections — Applies fuel quality corrections (MN<70: ×0.94, MN<80: ×0.98) and extreme altitude correction (>2000m: -0.5% per additional 1000m) on top of base fleet efficiency.
12
Voltage Recommendation — Auto: <10MW → 4.16kV, <50MW → 13.8kV, ≥50MW → 34.5kV. Or user manual value.
13
Transient Stability — transient_stability_check() verifies voltage sag ≤ 10%.
14
Availability Curve — calculate_availability_weibull() generates the availability curve over the project life with aging.
15
Emissions — calculate_emissions() computes NOx, CO, CO2, aftertreatment, urea, and 5 regulatory checks.
16
Footprint — calculate_footprint() calculates area by component including LNG storage if applicable. LNG sizing: lng_gal = lng_mmbtu_total / 0.075.
17
Net Efficiency — net_eff = fleet_eff × (1 - dist_loss%). Net efficiency delivered at point of use.
18
Financial Calculations — Full sequence: CAPEX (generators × $/kW × regional_mult × scale_factor + installation + CHP + BESS + aftertreatment), O&M (fixed + variable + labor + overhaul + BESS + AT + urea), fuel cost, carbon cost, MACRS depreciation, BESS repowering (batteries every 10 years, inverters every 15 years), LCOE, NPV, payback.
19
CAPEX Breakdown — Itemized: generation_units, installation_bop, tri_gen_chp, bess_system, aftertreatment.
20
O&M Breakdown — Itemized: fixed_kw_yr, variable_mwh, labor, overhaul, bess_om, aftertreatment_om, urea_def.
21
Gas Price Sensitivity — 20 points between 0 and 2× base price. Computes LCOE for each price. Finds breakeven gas price where LCOE = benchmark.
22
Grid Comparison — Cumulative costs: grid vs CAT (gross) vs CAT (with tax benefits). Determines crossover year.
23
Assemble Result — Packages all results into SizingResult (Pydantic model with 55+ typed fields).
